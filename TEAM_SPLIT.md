# Equal Team Split

## Teammate 1: Member + Booking Flow
Own these features:
- sign up / sign in UI
- profile and membership status page
- browse classes
- book class
- cancel booking
- booking history
- waitlist display

## Teammate 2: Staff/Admin + Check-In Flow
Own these features:
- QR check-in scan page
- check-in validation API
- trainer roster page
- mark attendance / no-show
- admin class CRUD
- membership plan CRUD
- basic reports dashboard

## Shared Rules
- Both use same MongoDB models
- Create separate feature branches
- Merge only after API testing
- Keep UI components reusable
